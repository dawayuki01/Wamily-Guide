## 1. Wamilyとは何か

### コアコンセプト
**「本棚のある、黒板がある、常連が集まる居酒屋」**

- 完成したサービスではなく、**育っていく過程そのものを見せる**場所
- オーナー（サワディー）の体温が随所に感じられる
- 来た人が自然と「話したくなる」空気がある
- 情報を届けるのではなく、**体験を届ける**

### Wamilyが届けたいもの
> 「自分たちの困りごとを解決した先に、誰かの困りごとが解決される。気づいたら本棚が増えていた。」

- 海外に行きたいと思っている家族の障壁を下げる
- みんなで障壁を下げ合うコミュニティ
- サワディー家が実際に行った国から始まり、有志が育てていく

### ターゲット
- 子連れで海外旅行が好き・行きたい日本人家族
- 最初はサワディーさんの周囲のコミュニティから手渡しで広がる
- 旅行者・現地在住者・これから行きたい人 すべてフラットに扱う

### オーナー情報
- **サワディー**（Wamilyオーナー）
- 子ども：3歳・5歳の男の子
- **家族で行った国（9カ国）：**
  - 🇬🇧 イギリス（ロンドン）
  - 🇫🇷 フランス（パリ）
  - 🇸🇪 スウェーデン（ストックホルム・マルメ）
  - 🇸🇬 シンガポール
  - 🇹🇭 タイ（バンコク）
  - 🇵🇭 フィリピン（マニラ）
  - 🇺🇸 アメリカ（LA・ハワイ）
  - 🇰🇷 韓国（ソウル）
  - 🇹🇼 台湾

## 4. トップページ仕様

### 構造
```
[のれん（ナビ）]
[ヒーロー：地球 + コピー + ピンアニメーション]
[オーナーコメント]
[ガイドブック本棚（横スクロール）]
[焚き火CTA]
[最近の動き（ライブフィード）]
```

### ヒーローセクション
- うっすら回転する地球（SVG、opacity: 0.1、40秒でゆっくり回転）
- 世界各地にピンが一個ずつ灯っていくアニメーション（遅延あり）
- 点線の飛行機軌跡（SVG、マスタードイエロー、opacity: 0.35）
- コピーはフェードアップで出現

**コピー（確定）：**
```
Family Travel Guide
家族の数だけ、旅がある。
誰かの経験が、誰かの地図になる。
```

### オーナーコメント
- マスタードイエローの左ボーダー
- サワディーアイコン（28px）が左に
- テキスト：「子どもが小さいころ、子連れ旅行の情報が全然見つからなくて困りました。信頼できる情報に出会えないあのストレス、わかります。だから作り始めました。まだ途中だし、間違いもあると思う。でも、だからこそ一緒に育てていきたい。」
- 署名：「— サワディー（Wamilyオーナー）　2024年1月から、少しずつ。」

### ガイドブック本棚
**重要：** 数字（掲載国数・スポット数など）は一切表示しない。

サブテキスト（本棚の上に）：
```
サワディー家が実際に行った国から作っています。行くたびに、増えていきます。
```

国カードの状態：
- `ちゃんと調べた`（孔雀色ライン）：台湾・イギリス
- `まだ旅の途中`（マスタードライン）：ハワイ・韓国・フランス・シンガポール・タイ・フィリピン・スウェーデン・LA
- `声を待ってる`（グレーライン、opacity 0.5）：将来追加予定の国

横スクロール形式。

### 焚き火CTA
```
🔥（揺れるアニメーション）
子連れで行った国、ありますか？
一言でいい。その経験が、次の誰かの背中を押します。
[旅のバトンを渡す ↗]（孔雀色ボタン）
```

### 最近の動き（ライブフィード）
- 孔雀色のパルスドット + 「最近の動き」ラベル
- Notionから自動取得・表示
- サワディーアイコン（マスタード）とユーザーアイコン（孔雀色）で区別

## 6. タブ詳細仕様

### タブ1：○○について
```
[リード文：「○○は、子どもと一緒に来ると、もっと好きになる街です。」]
[本文：親子視点でその国・都市の魅力を語る文章]
[親子ポイントカード × 4枚（孔雀色ライン）]
[基本データ：言語・通貨・時差・気候・電圧]
```

### タブ2：行く前に
トグル式アコーディオン。各セクションはデフォルト閉じ。

```
🏥 医療・安全（緊急連絡先・病院・薬局）
💰 お金・両替
🚇 交通・動き方
🧳 持ち物チェックリスト
```

**トグルの見た目：**
- セクションタイトル + サブテキスト（12px）
- 右に ▲ アイコン（開閉で回転）
- 開いた状態：情報ブロック + 💡ヒント枠

### タブ3：現地で楽しむ

#### 子連れスポットマップ
**3層構造のスポットリスト（重要）：**

```
🔴 いざという時（固定・全国共通）
  - 日本語/英語が通じる病院・小児科
  - 日本大使館
  - 薬局チェーン（Boots等）

🟢 ローカル体験（固定・国別）
  - スーパー（Waitrose、Tesco等）
  - マーケット

🟢 遊びに行く / 親子で食べる（固定・国別）
  - 無料博物館・公園等

🟡 可変スポット（自動更新）
  - Claude APIが定期的に更新
```

**4カテゴリーフィルター（確定）：**
```
すべて
🍽️ 親子で食べる
🎡 遊びに行く
🛒 現地の日常へ
🏥 いざという時
```

**表示ルール：**
- デフォルト：層ごとに表示、「遊びに行く」は3件表示 + 「もっと見る」
- フィルター選択時：全件表示、層ラベル非表示

**各スポットカードの要素：**
```
[アイコン] [スポット名]
[説明文（1〜2行）]
[● 営業中 / ● 要確認] [無料 / 有料] [確認日]
```

**営業状況バッジの色：**
- 営業中：緑（#edf6ec / #376e35）
- 要確認：アンバー（#fef8d8 / #7a6010）
- 閉業：グレー（非表示またはグレーアウト）

#### 今週のイベント
```
[点滅ドット] TimeOut London から自動取得　最終更新：今日

[イベントカード × 3件]
  - 日付・無料/有料バッジ
  - イベント名
  - 説明
  - 📍 場所

[TimeOut Londonでもっと見る →]
```

**注意：** 国によってイベント媒体が変わる（ロンドン=TimeOut London、パリ=TimeOut Paris等）

### タブ4：旅のバトン
```
📱 SNSで探す
  - #ロンドン子連れ（Instagram）
  - #ロンドン子連れ旅行
  - #ロンドン家族旅行
  - #london子連れ
  ※国に合わせてハッシュタグを変える

🎬 これ、見てみて（キュレーション）
  - オーナーが選んだYouTube・インスタ・ブログ
  - NotionのDBから取得
  - 「サワディーおすすめ」バッジ（マスタード）

💬 みんなの旅ばなし
  - 体験談カード（Notionから取得）
  - [旅のバトンを渡す ↗]（CTA）
  - ボタン下に小テキスト：「それが次に来る誰かの地図になります。」
```

## 7. 自動化仕様

### 全体の技術スタック
```
フロントエンド：HTML / CSS / JavaScript
ホスティング：GitHub Pages（無料）
コンテンツ管理：Notion API
自動化：Claude API + Make（Integromat）
地図：Google Maps API（埋め込み）
イベント：各国TimeOut / Visit系サイトのRSS
ドメイン：独自ドメイン（お名前.com等）
```

### Notion DB構造

#### 国別ガイドDB
```
国名 / 都市名 / ステータス（ちゃんと調べた / まだ旅の途中 / 声を待ってる）
/ 最終更新日 / 更新内容メモ
```

#### スポットDB
```
国名 / スポット名 / カテゴリ（親子で食べる / 遊びに行く / 現地の日常へ / いざという時）
/ 層（固定vital / 固定normal / 可変）
/ 説明 / 営業状況 / 料金（無料/有料） / 最終確認日 / Google Place ID
```

#### 最近の動きDB
```
投稿者 / 投稿者種別（オーナー / 在住者 / 旅行者）
/ 国名 / 本文 / 投稿日
```

#### キュレーションDB
```
国名 / タイプ（YouTube / Instagram / ブログ）
/ 名前 / 説明 / URL / 追加日
```

### 自動化パイプライン

#### 1. 営業状況の自動確認（週1回）
```
Make スケジュール → Notion スポットDB取得
→ Google Places API で営業状況確認
→ Notion に「営業中 / 要確認 / 閉業」を自動更新
→ 閉業の場合はスポットを非表示フラグ
```

#### 2. 最近の動きの自動生成
```
Notion ガイドページ更新を検知
→ Claude API が更新内容を要約（1〜2文）
→「○○を更新しました。△△のことが詳しくなりました。」
→ 最近の動きDB に自動追加
→ サイトに反映
```

#### 3. イベント情報の自動取得
```
Make スケジュール（週1回）
→ TimeOut London RSS取得
→ Claude API が子連れ関連性をフィルタリング
→ 関連性スコアの高いものを3〜5件選定
→ Notion イベントDBに保存
→ サイトに反映
```

#### 4. スポットの自動追加（オプション）
```
定期的にClaude APIが子連れスポット候補を検索
→ 候補をNotionに「要確認」ステータスで追加
→ サワディーさんが確認して承認 → 自動公開
```

## 13. UX体験設計の追記（2025年3月30日）

### コアUX体験
「探す行為が楽しい」Google Earth的な体験。
訪れた人が旅の主役になる。Wamilyは「場」と「体験」を提供する存在。

### トップページの体験フロー
```
地球が見える
　↓
サワディー家が行った9カ国だけピンが光っている
　↓
ピンをクリック / エリアを選ぶ
　↓
その国のガイドブックへ
　↓
更新されている匂い・信頼性のある情報・キュレーション
　↓
旅の準備が始まる
```

### オーナーの立ち位置（確定）
- トップページ：思想がデザインに溶け込む。押しつけない。

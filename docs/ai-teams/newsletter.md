# メルマガ部 運用ガイド

## 概要

週刊メルマガ配信 + ウェルカムメール（新規登録者向け）を管理する。

## Slackチャンネル

`#newsletter` — 配信結果レポート

## 配信スケジュール

| ワークフロー | スケジュール | 内容 |
|---|---|---|
| `newsletter.yml` | 毎週月曜 07:00 JST | 週刊キュレーションメール |
| `newsletter-announce.yml` | 毎日 08:00 JST | お知らせメール（予定日チェック） |
| `newsletter-sequence.yml` | 毎日 07:30 JST | ウェルカムメール |

## ウェルカムメール

### 設計思想

- 登録してくれた人は **Wamilyの思想に共感済みの仲間**
- 営業やナーチャリングではなく「ありがとう、一緒に育てていこう」のトーン
- **1通だけ**（登録翌日）。週刊メルマガに任せる

### フロー

1. 新規登録 → Notion購読者DBに「登録日」「シーケンスステップ=0」が設定される
2. 翌日 07:30 JST にスクリプトが実行
3. ウェルカムメールを1通送信
4. シーケンスステップ=999（完了）

### Phase 2 への展望

Wamilyが成長し、コミュニティやイベントが動き始めた段階で:
- 使い方ガイド（登録3日後）
- おすすめ紹介（登録7日後）
- 旅のバトン紹介（登録14日後）

を追加予定。`newsletter-sequence.js` のSEQUENCE配列にステップを追加し、`sequence-templates.js` にテンプレートを追加するだけで拡張可能。

### 安全策

- **1日1ステップ上限:** バースト送信防止
- **DRY_RUNモード:** `DRY_RUN=true` で実送信なしテスト可能
- **既存購読者:** シーケンスステップ=999 でスキップ

## トラブルシューティング

1. **配信失敗:** `#newsletter` のエラー通知を確認 → Resend APIキー・購読者DB確認
2. **ウェルカムが届かない:** 購読者の「登録日」がNullでないか確認
3. **テスト:** `DRY_RUN=true node scripts/newsletter-sequence.js` で確認
